# 작업 맥락 (Work Context)

이 파일은 테스트 자동화 툴 개발 과정에서 나누는 대화의 맥락을 기록하고 보존하기 위해 만들어졌습니다.

---

## 실행 방법 (How to Run)

**중요:** 기존의 `run.sh` 스크립트는 더 이상 사용하지 않습니다.

이제부터 애플리케이션은 `launcher.py`를 통해 실행합니다. 이 실행기는 최신 버전의 코드를 안전하게 동기화하고 메인 앱을 시작하는 역할을 합니다.

**로컬 PC 실행 순서:**

1.  터미널(또는 명령 프롬프트)을 엽니다.
2.  프로젝트 폴더로 이동합니다.
3.  `python3 launcher.py` 또는 `python launcher.py` 명령어를 입력하여 실행기를 시작합니다.
4.  나타나는 창에서 'Update & Launch QUATY' 버튼을 클릭합니다.

실행기는 자동으로 GitHub에서 최신 코드를 받아온 후, 메인 애플리케이션을 실행합니다.

## 미래 개발 로드맵 (Future Development Roadmap)

**최종 목표:** 사용자가 제공한 'QUATY Test Automation Tool' 스크린샷을 청사진으로 삼아, UI와 기능 면에서 동일한 수준의 전문적인 툴을 구현합니다. API 사용 효율을 높이기 위해 다음의 단계별 계획에 따라 체계적으로 개발을 진행합니다.

### Phase 1: UI 기반 구축

1.  **Dark Theme 적용:** 앱 전체에 'QUATY'와 유사한 어두운 테마의 스타일시트를 적용하여 시각적 정체성을 확립합니다. (**완료**)
2.  **레이아웃 재구성:** 'QUATY'의 레이아웃을 참조하여, 화면을 크게 **좌측 패널 (환경 설정, 실행 옵션 등)**, **중앙 패널 (URL, 스텝 테이블)**, **하단 패널 (실행 로그)**의 3단 구조로 재구성합니다. (**완료**)
3.  **위젯 재배치:** 기존의 위젯들을 새로운 레이아웃 구조에 맞게 재배치합니다. (**완료**)

### Phase 2: 핵심 기능 강화

1.  **Assertion Mode 구현:** 테스트 케이스에 '검증(assertion)' 스텝을 추가할 수 있는 기능을 구현합니다. (예: 특정 요소의 존재 여부, 텍스트 일치 여부 확인)
2.  **수동 스텝 추가 (+ Step):** 녹화 외에 사용자가 직접 스텝을 추가할 수 있는 기능을 구현합니다.
3.  **Headless 모드:** GUI 없이 백그라운드에서 테스트를 실행할 수 있는 옵션을 추가합니다.
4.  **스텝 선택 (Select) 기능:** 테이블에 체크박스를 추가하여 특정 스텝들을 선택하고 일괄 작업을 수행할 수 있는 기반을 마련합니다.

### Phase 3: 고급 기능 및 편의성 개선

1.  **CSV 데이터 연동:** 외부 CSV 파일의 데이터를 변수처럼 테스트에 주입하여 데이터 기반 테스트가 가능하도록 구현합니다.
2.  **All Batch 실행:** 파일 탐색기에서 여러 테스트 케이스를 선택하여 순차적으로 한 번에 실행하는 기능을 구현합니다.
3.  **환경 설정 (Environment):** 'Alpha', 'Real' 등 테스트를 실행할 서버 환경을 선택할 수 있는 옵션을 추가합니다.
4.  **Save Flow 기능:** 현재의 'Save Test' 기능을 대체하거나 보강하여, 더 직관적인 이름으로 테스트 흐름 전체를 저장하는 기능을 구현합니다.

## 대화 기록 (Dialogue History)

### 대화 1 ~ 10 (요약)
- 프로젝트 시작, Git 워크플로우 도입, 핵심 기능(녹화, 재생, 저장, 불러오기) 구현, 그리고 각종 치명적 오류(IndentationError, RecursionError)를 해결하며 다크 테마 적용 직전까지의 안정적인 기반을 마련함. 개발 효율성을 위해 'QUATY' 툴을 목표로 한 개발 로드맵을 수립함.

### 대화 11: UI 개선 착수 (Dark Theme 적용)

**USER:**
- 이전 오류들이 모두 해결되고 앱이 정상 실행됨을 확인.
- 로드맵에 따라 UI 개선 작업 착수를 요청.

**GEMINI:**
- '미래 개발 로드맵'을 `CONTEXT.md`에 상세히 기록.
- 로드맵 1단계에 따라 `stylesheet.qss` 파일을 생성하고, 앱 전체에 다크 테마를 적용.
- 'QUATY' 스크린샷을 참고하여 녹화/실행 버튼에 각각 빨간색/초록색을 부여.

**USER:**
- 다크 테마 구현을 긍정적으로 평가.
- 다만, 버튼 색상이 너무 밝다며 채도를 낮춰달라고 피드백.

**GEMINI:**
- 피드백을 반영하여 `stylesheet.qss`의 버튼 색상을 더 어둡고 차분한 톤으로 1차 수정.

**USER:**
- 수정된 색상이 더 낫지만, 채도를 조금 더 낮춰주기를 희망.
- 오늘은 여기까지 진행하고, 내일 이어서 작업할 수 있도록 맥락을 최종적으로 갱신해달라고 요청하며 마무리.

### 대화 12: 레이아웃 재구성 및 실행 편의성 개선

**USER:**
- 채도를 낮춘 버튼 색상 수정 및 3단 레이아웃 재구성 작업을 요청.
- `run.sh`를 매번 실행하는 것이 번거롭다며, 앱이 깨져도 안전하게 업데이트할 수 있는 더 편리한 실행 방법을 문의.

**GEMINI:**
- `stylesheet.qss`의 버튼 색상 채도를 추가 조정.
- `main.py`의 레이아웃을 '좌측(설정)-중앙(테이블)-하단(로그)' 구조로 재구성.
- 메인 앱과 분리된 별도의 '업데이트 실행기'(`launcher.py`)를 제안하고 구현. 이 실행기는 `git pull`로 코드를 최신화한 후 `main.py`를 실행시켜, 앱 자체에 오류가 발생해도 안전하게 업데이트할 수 있도록 함.
- 새로운 실행 방법을 `CONTEXT.md`에 문서화함.
