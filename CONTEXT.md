# 작업 맥락 (Work Context)

이 파일은 테스트 자동화 툴 개발 과정에서 나누는 대화의 맥락을 기록하고 보존하기 위해 만들어졌습니다.
대화의 흐름은 다음과 같은 형식으로 기록됩니다.

---

### 대화 1: 프로젝트 시작 및 기본 원칙 수립

**USER:**
- 테스트 자동화 툴 제작 프로젝트 시작을 제안.
- `CLAUDE.md` 행동 지침을 제공하며, 이를 모든 작업의 대전제로 삼을 것을 요청.

**GEMINI:**
- 제안에 동의하고, `CLAUDE.md` 파일 생성 및 내용 숙지.
- 해당 지침을 앞으로의 모든 작업에 적용할 것을 약속.

**USER_REACTION:**
- `CLAUDE.md` 생성에 긍정적인 반응.
- 다음 단계로, 대화의 맥락을 보존하기 위한 파일 생성을 요청.

---

### 대화 2: 테스트 자동화 툴 기본 구상 및 환경 제약 발견

**USER:**
- Python과 PySide6를 사용한 GUI 기반 테스트 자동화 툴의 기본 기능 명세를 설명.

**GEMINI:**
- 사용자의 기능 명세를 이해하고 `CONTEXT.md`에 기록.
- 첫 단계로 PySide6를 이용한 기본 GUI 뼈대(`main.py`) 생성을 제안.
- `main.py` 실행 시도 중 IDX 환경에서는 PySide6와 같은 GUI 라이브러리 실행이 불가능함을 인지.

**USER_REACTION:**
- IDX의 제약을 해결하기 위해 새로운 개발 워크플로우를 제안.

---

### 대화 3: 새로운 개발 워크플로우 수립 및 실행

**GEMINI:**
- 사용자의 제안에 동의하고 새로운 워크플로우를 `CONTEXT.md`에 기록.

**USER_REACTION:**
- Public GitHub 리포지토리를 생성하고 IDX 환경에 이미 연동했음을 알림.

**GEMINI:**
- 확인 후, 현재까지의 작업 내용을 커밋하기로 결정.

---
